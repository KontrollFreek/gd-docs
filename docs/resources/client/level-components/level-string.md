# Client Level String Resource

## Object String
The object string is contained within the inner level string. It is formatted as follows:

`{object};{object};{object};...`, where `object` is formatted as:

`{propertyKey},{propertyValue},{propertyKey},{propertyValue},...`.

E.g.:

`1,1,2,15,3,15,21,3,24,7;1,1,2,15,3,15,20,5,25,-1;` represents the following objects:

- Standard Block
  - ID: 1 (`1,1`)
  - X: 15 (`2,15`)
  - Y: 15 (`3,15`)
  - Main Color ID: 3 (`21,3`)
  - Z Layer: 7 (`24,7`)
- Standard Block
  - ID: 1 (`1,1`)
  - X: 15 (`2,15`)
  - Y: 15 (`3,15`)
  - Editor Layer 1: 5 (`20,5`)
  - Z Order: -1 (B3) (`25,-1`)

Notice that every single objects contains at least 3 properties, `1`, `2`, `3`, which represent the Object's ID, X position, and Y position respectively. Even in the case that the objects' X and Y positions are 0, the keys are not omitted. <!-- TODO: Verify -->

Property keys reflect the keys found in the following table, whereas property values are formatted as follows

| Type                | Formatting                                                                                  |
| ------------------- | ------------------------------------------------------------------------------------------- |
| **integer**         | Decimal numerical value string (e.g. `421`)                                                 |
| **bool**            | `1` if true, `0` if false                                                                   |
| **float**           | Decimal numerical value string, using `.` as the decimal point separator (e.g. `12.41`)     |
| **any enumeration** | Same as **integer**                                                                         |
| **integer array**   | All values are separated by `.` (e.g. `1.42.51.23` for `{ 1, 42, 51, 23 }`)                 |
| **HSV**             | `{h}a{s}a{v}a{s_checked}a{v_checked}`, where each name reflects the respective HSV property |
| **string**          | Raw string without quotation marks (e.g. `ExampleString`)                                   |

### Level String Data

> [!IMPORTANT]
> Unfortunately, due to there being 500+ possible property keys, not all are documented here. This list only contains properties found on most standard objects and some common triggers. For a more exhaustive list, you can visit [Flowvix's GD Info Explorer](https://flowvix.github.io/gd-info-explorer/props) for a full list of properties, and use the official [Geometry Dash Editor Guide](https://www.robtopgames.com/files/GDEditor.pdf) for a better understand of what each option may do.
